Title: ADR-0015: Email-Safe Report Generation with Inline Styles Only

Status: accepted

Date: 2026-04-22

## Context

Platform Compass delivers diagnostic results to customers and to the operator (Chris) as HTML email reports. Email clients have notoriously poor CSS support: Gmail strips `<style>` blocks in some contexts, Outlook uses the Word rendering engine (no CSS Grid, no Flexbox), and Apple Mail is the most permissive but still benefits from inline styles.

Two implementation approaches were considered:

**Option A — React components rendered to HTML string (React DOM Server):** Report is a React component styled with Tailwind or CSS Modules, rendered via `renderToStaticMarkup`. Cleaner authoring but produces class-based styles that are stripped by most email clients. Requires a CSS inliner post-processing step (e.g. `juice`) to be email-safe.

**Option B — Pure JS template with inline styles baked in:** Report is built as a plain JavaScript string template in `lib/compassReports.js`. All styles are applied as inline `style` attributes or within `<style>` blocks at the top of the email. No external CSS, no web fonts, no images hosted externally (brand colours are hex constants in the module). Works in all email clients without post-processing.

Option B was chosen because it eliminates the CSS inliner dependency, keeps the report generator framework-agnostic (consistent with ADR-0011), and guarantees email safety at authoring time rather than relying on a build-time transformation.

## Decision

Compass HTML reports are generated by `lib/compassReports.js` using inline `style` attributes and a top-level `<style>` block. No Tailwind classes, no external CSS files, no web fonts, and no externally hosted images are used in report HTML. Brand colour values are defined as constants within the module.

## Consequences

- Report styling changes require editing hex constants and inline style strings in `compassReports.js` — there is no shared token system with the Tailwind config (by design; see ADR-0003 for the Tailwind exception note).
- Table-based layouts are used for email structure — CSS Grid and Flexbox are avoided for Outlook compatibility.
- The module exports two functions: `buildSimpleReport` (customer-facing) and `buildAdvancedReport` (internal copy for the operator) — both must remain email-safe.
- When validating report changes, test output in at least Gmail (web), Outlook desktop, and Apple Mail before deploying.
- Use `scripts/test-compass-webhook.mjs --dry-run` to inspect generated report HTML without sending a live email.
